# Slicing Step

Create a TDD execution plan that links implementation slices to requirements,
acceptance criteria, expected touchpoints, red/green commands, verification, and
review focus.

Slices are a task graph. Each slice must expose complexity, dependency notes,
critical-path status, parallelizability, file ownership, conflict risk, test
strategy, and independent verification. Use this metadata to find bottlenecks
and avoid unsafe parallel work.
