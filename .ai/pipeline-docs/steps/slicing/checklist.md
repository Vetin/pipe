# Slicing Checklist

- Link every slice to requirements and acceptance criteria.
- Include red command, expected failure, green command, and verification
  commands.
- Include iteration budget, rollback point, independent verification, and
  failure triage notes.
- Include complexity, critical path, parallelizable marker, file ownership,
  conflict risk, dependency notes, and test strategy.
- Validate dependency chain and identify bottlenecks before implementation.
- Use expected touchpoints instead of allowed or forbidden file enforcement.
- Record `Docs Consulted: Slicing` in `execution.md`.
- Keep implementation blocked until readiness is approved or delegated.
