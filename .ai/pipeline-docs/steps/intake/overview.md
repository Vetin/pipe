# Intake Step

Create the feature run, branch, worktree, workspace, routing files, initial
execution log, and non-delegable checkpoints. Intake decides whether the request
is clear enough to proceed; it does not write product, architecture, design, or
implementation code.

Classify rigor as minimal, standard, or comprehensive. Score ambiguity across
functional scope, data, UX, integrations, security/RBAC, edge cases,
non-functional constraints, and completion signals. Stop before downstream
planning when ambiguity is unsafe, or when destructive, security, public API, or
data ownership questions are unresolved.
