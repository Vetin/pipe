# Technical Design Checklist

- Load the technical design docset.
- Read feature contract and architecture.
- Carry architecture change delta into implementation deltas.
- Define modules and responsibilities.
- Define dependency, ownership, critical-path, parallelization, and conflict-risk
  expectations for slices.
- Define contracts and data model when relevant.
- Define error handling, security considerations, tests, migration, rollback, and
  integration notes.
- Record decision traceability from requirements and ADRs to design choices.
- Record `Docs Consulted: Technical Design` in `execution.md`.
