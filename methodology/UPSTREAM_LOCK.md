# Upstream Lock

The current pipeline is synthesized from these methodology families:

| Source family | Native use |
| --- | --- |
| BMAD-style role gates | Planning, architecture, review, and finish checkpoints |
| Spec Kit style specs | Feature contract, acceptance criteria, and traceability |
| OpenSpec style changes | Change-scoped artifacts, contracts, and promotion safety |
| Superpowers style execution loops | TDD evidence, iteration ledgers, and resume checkpoints |
| Agent Skills guidance | Focused skill bodies, progressive disclosure, bundled references |
| EARS/constitutional SDD | Testable requirements, non-negotiable safety rules |

Vendored upstream snapshots are intentionally not required for normal pipeline
execution. If upstream docs are vendored later, lock commit SHAs and license
notes here before using them as authoritative references.
